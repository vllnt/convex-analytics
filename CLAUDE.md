
## Convex Component Pattern

- **Schema is truth** ‚Äî `schema.ts` defines all tables and indexes. Every query/mutation references these.
- **Never edit `_generated/`** ‚Äî these files are auto-generated by Convex codegen.
- **Component isolation** ‚Äî `defineComponent("analytics")` creates a namespace. Child components (`aggregate`, `shardedCounter`, `rateLimiter`) are mounted via `component.use()`.
- **Consumer mounting** ‚Äî users call `app.use(analytics)` in their `convex.config.ts`, then `new ConvexAnalytics(components.analytics)`.

## Key Patterns

- **Sharded counter** ‚Äî 16-shard counter for high-throughput `track()` writes. `count()` reads are O(shards) not O(n).
- **Rate limiter** ‚Äî token bucket per sessionId (100/min, burst 10). Silently drops excess events (returns null, no error).
- **Aggregate** ‚Äî `DirectAggregate` with namespace `"eventName:YYYY-MM-DD"` for O(log n) date-range counts.
- **Session lifecycle** ‚Äî auto-created on first event per sessionId. Closed by cron after 30min inactivity. `seqNum` derived from session's `eventCount` (TOCTOU-safe).
- **Daily rollups** ‚Äî cron runs every 5min, aggregates last 10min of events into `daily_rollups` with pre-computed dimension breakdowns. Idempotent (uses `Math.max` merge).
- **Alias** ‚Äî paginated reassignment (500/batch) of events+sessions from anonymousId to identifiedId, then user record merge.

## Common Pitfalls

1. **Don't `.collect()` the events table** ‚Äî it can have millions of rows. Use indexes + `.take(limit)`.
2. **Event name validation** ‚Äî must match `/^[a-zA-Z_][a-zA-Z0-9_]{0,63}$/`. Reject at track() and API layer.
3. **Property schema filtering** ‚Äî when `event_schemas` entry exists, unknown properties are silently dropped, and type mismatches are silently dropped. This is intentional (not an error).
4. **Rollup idempotency** ‚Äî uses `Math.max` not `+=` when merging into existing rollup rows. Safe to re-run.
5. **Counter drift** ‚Äî `rebalance` cron detects >1% drift between sharded counter and actual event count. Log warning only.

## Testing

Uses `convex-test` with Vitest. Test files in `packages/convex-analytics/tests/`.

```typescript
import analyticsTest from "convex-analytics/test";
import { convexTest } from "convex-test";

function initTest() {
  const t = convexTest();
  analyticsTest.register(t);
  return t;
}
```
